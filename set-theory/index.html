Troubleshooting MIDI

  • MIDI N/A or blocked: You're likely opening the file directly (file://). Use a local server — run python3 -m http.server 8000 in the folder containing this file, then visit http://localhost:8000
  • No devices found: Make sure your MIDI device is connected and powered on before loading the page. Try refreshing.
  • Permission denied: Check your browser settings to allow MIDI access for this site. You may need to click the lock icon in the address bar.
  • Chrome/Edge recommended: Web MIDI has best support in Chromium-based browsers. Firefox requires enabling dom.webmidi.enabled in about:config.
  • Notes not registering: Check that you've selected the correct MIDI input device from the dropdown.

Academic Analysis Tools

The Academic tab in the right panel provides advanced features for scholarly analysis, inspired by published set-theory analyses like those in Perspectives of New Music.

Ordered Segment-Class Mode

Traditional set theory treats pitch class sets as unordered collections. For row and segment analysis, enable Ordered Mode to preserve input order.

  • X (Original): The ordered sequence as entered (e.g., G C A♭ B♭ B A)
  • R(X): Retrograde — reversed order
  • I(X): Inversion — reflected around the first note
  • RI(X): Retrograde-Inversion — both transformations combined
Try It

Enter G C Ab Bb B A (Wuorinen's String Trio hexachord X), then check "Treat as ordered row" in the Academic tab to see all four row forms.

Rotational Arrays

Rotational arrays display all cyclic rotations of an ordered segment. These tables can be read in multiple directions:

  • → Left to Right: Transposition
  • ← Right to Left: Retrograde
  • ↓ Top to Bottom: Inversion
  • ↑ Bottom to Top: Retrograde-Inversion

Click M₅(X) Array or MI₇(X) Array to see the multiplicative transformations.

M-Related Set Classes

The M-transformation maps pitch classes by multiplication modulo 12:

  • M₅ (circle of fourths): Maps C→C, C#→F, D→B♭, etc. Also called "chromatic to quartal" mapping.
  • M₇ (circle of fifths): The inverse of M₅, maps onto the ascending fifths cycle.

Some sets are M-invariant — they map to themselves under M₅ or M₇. This is shown in the display.

Section Timeline

Track pitch-class material across sections of a composition:

  1. Enter a pitch class set for a section

The timeline shows set content, Forte number, and preserves ordered sequences if enabled.

Tonnetz Lattice

The Tonnetz (German: "tone network") visualizes harmonic relationships in Just Intonation. Each node represents a pitch as a ratio relative to the tonic (1/1).

Lattice Axes

  • Horizontal (→): Perfect fifths (×3/2 going right)
  • Vertical (↑): Major thirds (×5/4 going up)
  • Depth (7-limit): Septimal sevenths (×7/4)
  • 4th Dimension (11-limit): Undecimal ratios (×11/8)

Combination Product Sets (CPS)

CPS structures (invented by Erv Wilson) create symmetrical pitch collections with no tonal center:

  • Hexany: 4 factors taken 2 at a time = 6 notes
  • Dekany: 5 factors taken 2 at a time = 10 notes
  • Eikosany: 6 factors taken 3 at a time = 20 notes

Select a CPS preset or create custom ones in the CPS tab or lattice controls.

What is Pitch Class Set Theory?

Pitch class set theory is a method for analyzing music that doesn't follow traditional tonal harmony. Developed primarily by Allen Forte in the 1970s (building on work by Milton Babbitt and others), it provides systematic tools for understanding atonal, twelve-tone, and post-tonal music by composers like Schoenberg, Webern, Berg, Bartók, and Stravinsky.

Core Concepts

  • Pitch Class (PC): A note regardless of which octave it's in. All C's in every octave = PC 0. Uses integers 0-11 (or note names C through B).
  • PC Set: An unordered collection of pitch classes, written in braces like {0, 4, 7}. Order doesn't matter—{0, 4, 7} = {7, 0, 4}.
  • Cardinality: The number of pitch classes in a set. A trichord has 3, tetrachord has 4, pentachord has 5, hexachord has 6.
  • Normal Form: The most compact way to arrange a set clockwise on the pitch class clock, starting from the "smallest span" position.
  • Prime Form: The most reduced form—normal form transposed to start on 0, then compared with its inversion to find the most packed version.

Operations (Transformations)

  • Transposition (Tn): Add n to each pitch class (mod 12). T5{0,4,7} = {5,9,0}.
  • Inversion (In): Subtract each PC from n (mod 12). Maps upward motion to downward motion.
  • TnI: Invert first, then transpose. The compound operation TnI.
  • Multiplication (Mn): Multiply each PC by n (mod 12). M5 and M7 are particularly useful.

Forte Classification System

Allen Forte catalogued all possible set classes (equivalence classes under transposition and inversion) and assigned them numbers in the format C-N where C is cardinality and N is an ordering number.

  • 3-11: The minor triad and major triad (they're inversionally equivalent)
  • 4-27: The dominant seventh / half-diminished seventh
  • 6-Z44: The "Z" indicates a Z-related pair (same interval vector, different sets)

Interval Vector

A 6-digit vector counting how many of each interval class appear in a set:

  • IC1: minor 2nd / major 7th (1 or 11 semitones)
  • IC2: major 2nd / minor 7th (2 or 10 semitones)
  • IC3: minor 3rd / major 6th (3 or 9 semitones)
  • IC4: major 3rd / minor 6th (4 or 8 semitones)
  • IC5: perfect 4th / perfect 5th (5 or 7 semitones)
  • IC6: tritone (6 semitones—its own complement)

Example: Major triad [0,4,7] has vector <001110> — one minor 3rd (4-7), one major 3rd (0-4), one perfect 5th (0-7).

Z-Relation

Two set classes are Z-related if they share the same interval vector but aren't related by transposition or inversion. They sound similar in interval content but have different structures. About 15% of set classes have a Z-partner.

Invariance Vectors

Invariance vectors (8 positions) reveal symmetry properties of a set:

Positions 1-4: Self-mapping (set maps into itself)
  • Pos 1: Tn invariance — how many transpositions map set to itself
  • Pos 2: TnI invariance — transposition + inversion self-mapping
  • Pos 3: TnM invariance — transposition + M5 self-mapping
  • Pos 4: TnMI invariance — transposition + M5 + inversion
Positions 5-8: Complement mapping (set maps to its complement)
  • Pos 5-8: Same operations but mapping to complement

Higher values in positions 1-4 indicate more symmetry. Higher values in positions 5-8 indicate less symmetry. Highly symmetric sets like the augmented triad [048] or whole-tone scale [024681t] have high invariance.

M-Transformations

The M transformation (multiplication) maps pitch classes by multiplying by 5 or 7 (mod 12):

  • M₅: Circle of fourths mapping — preserves interval class but reorders
  • M₇: Circle of fifths mapping — inverse of M₅
  • MI₅/MI₇: Multiplication combined with inversion

Example: M₅ of set 6-1 [012345] = set 6-32 [024579]. They have identical interval vectors but different prime forms.

Why Use Set Theory?

Traditional analysis fails for music without keys, triads, or functional harmony. Set theory reveals:

  • Hidden relationships between seemingly unrelated passages
  • Compositional techniques (how a piece is constructed from motivic cells)
  • Structural coherence in atonal works
  • Voice-leading and transformation patterns

Just Intonation Basics

Just Intonation (JI) uses simple frequency ratios instead of the compromises of equal temperament:

  • 3-limit (Pythagorean): Only uses ratios with factors of 2 and 3
  • 5-limit: Adds factor of 5 for pure major/minor thirds
  • 7-limit: Adds factor of 7 for the "harmonic seventh"
  • Higher limits: 11, 13, etc. for increasingly exotic intervals

The Lattice Builder visualizes these relationships as a geometric space where horizontal = fifths (3/2), vertical = thirds (5/4), and depth = septimal intervals (7/4).

Key JI Ratios

  • 2/1 = 1200¢ — Octave
  • 3/2 = 702¢ — Perfect fifth
  • 4/3 = 498¢ — Perfect fourth
  • 5/4 = 386¢ — Major third (pure)
  • 6/5 = 316¢ — Minor third (pure)
  • 7/4 = 969¢ — Harmonic/natural seventh
  • 9/8 = 204¢ — Major second (Pythagorean)

#!/usr/bin/env python3
# WebSocket to OSC Bridge - pip install websockets python-osc
import asyncio, json, websockets
from pythonosc import udp_client

WS_PORT, SC_PORT = 8765, 57120
osc = None

async def handle(ws, path=None):
    global osc
    if not osc: osc = udp_client.SimpleUDPClient("127.0.0.1", SC_PORT)
    print(f"Client connected")
    try:
        async for msg in ws:
            data = json.loads(msg)
            osc.send_message(data.get('address',''), data.get('args',[]))
            await ws.send(json.dumps({'status':'ok'}))
    except: pass

async def main():
    print(f"Bridge running - WS:{WS_PORT} -> OSC:{SC_PORT}")
    async with websockets.serve(handle, "localhost", WS_PORT):
        await asyncio.Future()

if __name__ == "__main__":
    asyncio.run(main())
